Roadmap

Promote native authority one gate at a time.

The vault recommends not jumping from advisory snapshots to full native authority. The safe next milestone is a native memory proposal gate that remains bounded and testable.

S1

Native Advisory Snapshot

Already present: Python can see native state, but native cannot execute tools or bypass Python policy.

S2

Native Memory Proposal Gate

Next target: expose native memory candidates, route them through a Python gate, persist accepted memories, and return commit feedback.

S3

Mode and Drive Influence

Native work mode begins influencing behavior mode and prompt context without granting direct execution authority.

S4

Permission and Lifecycle Authority

Only after memory gates are stable, native advisory approval can become part of selected permission and daemon lifecycle decisions.
